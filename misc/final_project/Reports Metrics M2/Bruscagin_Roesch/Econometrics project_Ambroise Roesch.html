Defining parameters and variables

Our first variable is the “disaster_damages” one. We assume that disaster damages (as % of GDP) follow a right-skewed distribution (most disasters are small, but some are extreme). In fact, most events have relatively minor impacts, but a few extreme events cause significant damage (e.g., the 2011 Japan tsunami causing 5.98% of GDP damage). It has a mean of 0.1 as it is consistent with the examples in the paper where the average qualifying disaster was estimated at around 0.19% of GDP for events in the dataset. A skewness of 2 was chosen to approximate the variability and skewness observed in disaster costs as presented in the paper’s dataset.

By incorporating credit growth, the simulation tests the paper’s hypothesis that its interaction with disaster damages significantly impacts financial stability. The assumption about credit growth distribution was made in the simulation as a reasonable simplification, based on the idea that credit growth in economies typically fluctuates symmetrically around a mean value.

n <- 100

mean_damages <- 0.1 
skewness <- 2

disaster_damages <- rgamma(n, shape = 2, scale = mean_damages / skewness)

mean_credit_growth <- 0.03 # this is our average annual credit growth (3%)
sd_credit_growth <- 0.02 

credit_growth <- rnorm(n, mean_credit_growth, sd_credit_growth)

Crisis risk model

Assumption: Crisis risk increases with disaster damages and credit growth. We model it as a probability based on a logistic function.

The crisis risk parameters in the simulation are based on reasonable approximations aligned with the paper’s findings. The intercept (alpha=−3) sets the baseline crisis probability at approximately 4.7%, reflecting the low but non-negligible annual likelihood of financial crises historically. The coefficient for disaster damages (beta_damages =10) ensures that damages equivalent to 0.1% of GDP increase crisis probability significantly, consistent with the paper’s estimated 6–18% increase in crisis odds. The coefficient for credit growth (beta_credit =5) models its amplifying effect, reflecting findings that rapid credit expansion elevates financial instability.

alpha <- -3 
beta_damages <- 10
beta_credit <- 5
sigma_u <- 1.5

We compute the log-odds of crisis risk to model the relationship between disaster damages, credit growth, and the likelihood of a financial crisis. This approach follows the logistic regression framework. These log-odds are then transformed into probabilities using the logistic function.

log_odds_crisis <- alpha + beta_damages * disaster_damages + beta_credit * credit_growth + sigma_u
crisis_probability <- 1 / (1 + exp(-log_odds_crisis))

The crisis outcomes are simulated as a binary variable using these probabilities, reflecting whether a crisis occurred in a given period.

crisis <- rbernoulli(n, crisis_probability)

The log-odds of a crisis occurring when both disaster_damages and credit_growth are zero is -1.5388. Converting log-odds to probability, this means the baseline probability of a crisis, when no damages or credit growth are present, is approximately 17,6%.

The coefficient for disaster_damages is 10,3042. This is a very strong positive effect, which means that if disaster damages increase by 0.1, the log-odds of a crisis increase by 1,03042, which corresponds to multiplying the odds of a crisis by more or less 2,8. This large coefficient highlights the substantial role that disaster damages play in elevating crisis risk.

The coefficient for credit_growth is -0.5298, which means that for every one-unit increase in credit growth, the odds of a crisis is reduced by a factor of approximately 0,588. But since credit growth is typically much smaller than one in practical terms, the overall impact of credit growth on crisis probability may be modest.

Repeat

To understand what might happen with a different sample, we rerun the analysis recompiling the code multiple times. Each time, this generates a new dataset and re-estimates the values using the new data. To streamline this process, we use a for loop to automatically repeat the analysis and store the results.

n_iter <- 100 
res <- NULL 

set.seed(42)

for (i in 1:n_iter) {

  #à chaque itération, génération des mêmes données
  disaster_damages <- rgamma(n, shape = 2, scale = mean_damages / skewness)
  credit_growth <- rnorm(n, mean_credit_growth, sd_credit_growth)
  log_odds_crisis <- alpha + beta_damages * disaster_damages + beta_credit * credit_growth
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbernoulli(n, crisis_probability)
  u <- rnorm(n, 0, sigma_u) 
  
  data_sim <- tibble(    # création tableau des données simulées
    disaster_damages,
    credit_growth,
    crisis,
    u
  )
  
  reg <- glm(crisis ~ disaster_damages + credit_growth, data = data_sim, family = binomial) #à chaque itération, régression logistique
  
  coef_table <- broom::tidy(reg) # extraction des coeffs
  conf_intervals <- confint(reg) #intervalles de confiance
  
  coef_table <- coef_table |> 
    mutate(
      conf.low = conf_intervals[, 1],# borne inf
      conf.high = conf_intervals[, 2] # borne sup
    )
  
  # garder disaster + credit
  coef_table <- coef_table |> 
    filter(term %in% c("disaster_damages", "credit_growth")) |> 
    mutate(sim_id = i)
  # ajour au tableau général qui compile les résultats
  res <- bind_rows(res, coef_table)
}

We create a function to generate data and run it on our set of parameters.

generate_data <- function(n,
                          alpha,
                          beta_damages,
                          beta_credit,
                          mean_damages,
                          skewness,
                          mean_credit_growth,
                          sd_credit_growth,
                          sd_error) {
  disaster_damages <- rgamma(n, shape = 2, scale = mean_damages / skewness)
  credit_growth <- rnorm(n, mean_credit_growth, sd_credit_growth)
  error_term <- rnorm(n, mean = 0, sd = sd_error)
  log_odds_crisis <- alpha + beta_damages * disaster_damages + beta_credit * credit_growth + error_term
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbernoulli(n, crisis_probability)
  tibble(disaster_damages, credit_growth, crisis)
}

We create a function that takes as input the tibble generated by generate_data(), runs the estimation and selects the information of interest.

run_estim <- function(data) {
  glm(crisis ~ disaster_damages + credit_growth, data = data, family = binomial) |> 
    broom::tidy() |> 
    filter(term %in% c("disaster_damages", "credit_growth"))
}

The compute_sim function generates simulated data based on specified model parameters, runs a logistic regression on the data, and returns the estimated results along with the simulation parameters.

compute_sim <- function(n, alpha, beta_damages, beta_credit, mean_damages, skewness, mean_credit_growth, sd_credit_growth, sd_error) {
  data <- generate_data(
    n = n,
    alpha = alpha,
    beta_damages = beta_damages,
    beta_credit = beta_credit,
    mean_damages = mean_damages,
    skewness = skewness,
    mean_credit_growth = mean_credit_growth,
    sd_credit_growth = sd_credit_growth,
    sd_error = sd_error
  )
  
  results <- run_estim(data)
  
  results |> 
    mutate(
      n = n,
      alpha = alpha,
      beta_damages = beta_damages,
      beta_credit = beta_credit,
      mean_damages = mean_damages,
      skewness = skewness,
      mean_credit_growth = mean_credit_growth,
      sd_credit_growth = sd_credit_growth,
      sd_error = sd_error
    )
}

The credit and disaster damages interactions model

We want know to evaluate the interaction between disaster damages and credit growth in predicting financial crises in order to test whether or not their combined effects significantly amplify crisis risk.

We simulate a dataset for a logistic regression model where the binary outcome (crisis) depends on disaster_damages, credit_growth, their interaction, and a control variable (control_var), along with random noise (error_term).

generate_data <- function(n,
                          alpha,
                          beta_damages,
                          beta_credit,
                          beta_interaction,
                          beta_control,
                          mean_damages,
                          skewness,
                          mean_credit_growth,
                          sd_credit_growth,
                          mean_control,
                          sd_control,
                          sd_error) {
  disaster_damages <- rgamma(n, shape = 2, scale = mean_damages / skewness)
  credit_growth <- rnorm(n, mean_credit_growth, sd_credit_growth)
  control_var <- rnorm(n, mean_control, sd_control)
  error_term <- rnorm(n, mean = 0, sd = sd_error)
  log_odds_crisis <- alpha +
    beta_damages * disaster_damages +
    beta_credit * credit_growth +
    beta_interaction * disaster_damages * credit_growth +
    beta_control * control_var +
    error_term
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbernoulli(n, crisis_probability)
  tibble(
    disaster_damages,
    credit_growth,
    control_var,
    crisis
  )
}

We generate data using the specified parameters, fitting the logistic regression model to estimate the effects of predictors and their interactions on the binary outcome (crisis), and returns the regression results along with the parameters used for the simulation.

compute_sim <- function(n, alpha, beta_damages, beta_credit, beta_interaction, beta_control, 
                        mean_damages, skewness, mean_credit_growth, sd_credit_growth,

    n = n,

    beta_damages = beta_damages,
    beta_credit = beta_credit,

We simulate a dataset by generating time-dependent variables (disaster damages, credit growth, control variables) over four time periods. And calculate the log-odds of the binary crisis outcome based on these variables and their interactions. We output the simulated data in a structured tibble.

# generate data for the simulation
generate_time_series_data <- function(n, alpha, beta_damages, beta_credit, beta_interaction, beta_control, mean_damages, skewness, mean_credit_growth, sd_credit_growth, sd_control, sd_error) {
  
  # Generate time-dependent disaster damages and credit growth
  disaster_damages <- matrix(rgamma(n * 4, shape = 2, scale = mean_damages / skewness), nrow = n, ncol = 4)
  credit_growth <- matrix(rnorm(n * 4, mean_credit_growth, sd_credit_growth), nrow = n, ncol = 4)
  control_var <- matrix(rnorm(n * 4, 0, sd_control), nrow = n, ncol = 4)
  
  # log-odds et crisis probabilities
  log_odds_crisis <- alpha +
    beta_damages[1] * disaster_damages[, 1] + beta_damages[2] * disaster_damages[, 2] + beta_damages[3] * disaster_damages[, 3] + beta_damages[4] * disaster_damages[, 4] +
    beta_credit[1] * credit_growth[, 1] + beta_credit[2] * credit_growth[, 2] + beta_credit[3] * credit_growth[, 3] + beta_credit[4] * credit_growth[, 4] +
    beta_interaction[1] * (disaster_damages[, 1] * credit_growth[, 1]) +
    beta_interaction[2] * (disaster_damages[, 2] * credit_growth[, 2]) +
    beta_interaction[3] * (disaster_damages[, 3] * credit_growth[, 3]) +
    beta_interaction[4] * (disaster_damages[, 4] * credit_growth[, 4]) +
    beta_control[1] * control_var[, 1] + beta_control[2] * control_var[, 2] + beta_control[3] * control_var[, 3] + beta_control[4] * control_var[, 4] +
    rnorm(n, mean = 0, sd = sd_error)
  
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbinom(n, 1, crisis_probability)
  
  # tibble avec generated data
  tibble(
    disaster_damages_t = disaster_damages[, 1],
    disaster_damages_t_1 = disaster_damages[, 2],
    disaster_damages_t_2 = disaster_damages[, 3],
    disaster_damages_t_3 = disaster_damages[, 4],
    credit_growth_t = credit_growth[, 1],
    credit_growth_t_1 = credit_growth[, 2],
    credit_growth_t_2 = credit_growth[, 3],
    credit_growth_t_3 = credit_growth[, 4],
    control_var_t = control_var[, 1],
    control_var_t_1 = control_var[, 2],
    control_var_t_2 = control_var[, 3],
    control_var_t_3 = control_var[, 4],
    crisis = crisis
  )
}

We introduce a function to generate time-series data based on specified parameters and fit a logistic regression model to estimate the relationships between disaster damages, credit growth, their interactions, and control variables on the likelihood of a crisis.

# fonction
simulate_time_series <- function(n, alpha, beta_damages, beta_credit, beta_interaction, beta_control, mean_damages, skewness, mean_credit_growth, sd_credit_growth, sd_control, sd_error) {
  
  # generated data
  data <- generate_time_series_data(
    n = n,
    alpha = alpha,
    beta_damages = beta_damages,
    beta_credit = beta_credit,
    beta_interaction = beta_interaction,
    beta_control = beta_control,
    mean_damages = mean_damages,
    skewness = skewness,
    mean_credit_growth = mean_credit_growth,
    sd_credit_growth = sd_credit_growth,
    sd_control = sd_control,
    sd_error = sd_error
  )
  
  # logistic regression
  model <- glm(
    crisis ~ disaster_damages_t + disaster_damages_t_1 + disaster_damages_t_2 + disaster_damages_t_3 +
      credit_growth_t + credit_growth_t_1 + credit_growth_t_2 + credit_growth_t_3 +
      I(disaster_damages_t * credit_growth_t) + I(disaster_damages_t_1 * credit_growth_t_1) +
      I(disaster_damages_t_2 * credit_growth_t_2) + I(disaster_damages_t_3 * credit_growth_t_3),
    data = data,
    family = binomial
  )
  
  # retour résultats
  tidy(model)
}

Test for omitted variables : full model

In this simulation, we model the effect of disaster damages and credit growth on the probability of a financial crisis, including both variables in the regression. Credit growth is simulated as a function of disaster damages plus an error term, reflecting how disasters can influence financial conditions. This simulation tests the independent effects of both variables while accounting for their interaction. The goal was to validate the importance of including both factors, as hypothesized in the paper, and to observe how they jointly impact crisis risk.

generate_data <- function(n, alpha, beta_damages, beta_credit, sd_error) {
  disaster_damages <- rgamma(n, shape = 2, scale = 0.1)
  credit_growth <- 0.02 + 0.5 * disaster_damages + rnorm(n, mean = 0, sd = sd_error)
  log_odds_crisis <- alpha +
    beta_damages * disaster_damages +
    beta_credit * credit_growth
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbernoulli(n, crisis_probability)
  tibble(
    disaster_damages,
    credit_growth,
    crisis
  )
}

Test for omitted variables : fiscal capacity

This simulation extends the model by including fiscal capacity as a mitigating factor in the relationship between disaster damages, credit growth, and crisis probability. Fiscal capacity is simulated first as a normally distributed variable, influencing credit growth and directly reducing crisis risk. The interaction between disaster damages and credit growth is also included. This model tests whether fiscal capacity plays a significant role in buffering against financial crises.

generate_data <- function(n,
                          alpha,
                          beta_damages,
                          beta_credit,
                          beta_control,
                          mean_damages,
                          skewness,
                          h_0,
                          h_1,
                          sd_credit_shock,
                          sd_fiscal_capacity,
                          mean_control,
                          sd_control,
                          sd_error) {
  disaster_damages <- rgamma(n, shape = 2, scale = mean_damages / skewness)
  
  raw_fiscal_capacity <- rgamma(n, shape = 2, scale = sd_fiscal_capacity)
  fiscal_capacity <- raw_fiscal_capacity / (max(raw_fiscal_capacity) + 0.01)
  
  credit_growth <- h_0 + h_1 * fiscal_capacity + rnorm(n, 0, sd_credit_shock)
  
  control_var <- rnorm(n, mean_control, sd_control)
  
  error_term <- rnorm(n, mean = 0, sd = sd_error)
  
  log_odds_crisis <- alpha +
    beta_damages * disaster_damages +
    beta_control * control_var +
    error_term
  
  crisis_probability <- 1 / (1 + exp(-log_odds_crisis))
  crisis <- rbernoulli(n, crisis_probability)
  
  tibble(
    disaster_damages,
    fiscal_capacity,
    credit_growth,
    control_var,
    crisis
  )
}
